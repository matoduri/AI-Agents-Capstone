# Agent Spec: Newborn Tracker Workflow

## Overview
A conversational assistant built with ADK 2.0 using the new graph Workflow API. It tracks multiple newborns' profiles and their daily health statistics (weight, feedings, diaper changes) based on parent messages. It maintains state in-memory across the session.

## Example Use Cases
- **Newborn Profiling**: Parent says "I want to track baby Liam." The agent creates Liam's profile.
- **Clarification**: If the parent says "Liam drank 3 oz" but the tracker knows about both "Liam" and "Noah", it confirms: "Are we tracking stats for Liam?" If the name is unknown or ambiguous, it asks the user to confirm.
- **Daily Stats Entry**: Parent says "Liam had a dirty diaper" or "Emma weighed 7 lbs 8 oz." The agent records the information under that newborn's statistics.

## Core Capabilities & Graph Structure
The workflow will consist of a directed graph:
1. **START**: Receives the raw parent input.
2. **Extractor Agent/Function**: An LLM-powered or function-based node that processes the input to extract newborn names, weights, feedings, and diaper events.
3. **Newborn Resolver**: Verifies which newborn is targeted. If ambiguous (multiple match, no active profile select, or new newborn profile needed), it yields a `RequestInput` to ask the parent.
4. **State Updater**: Updates the in-memory state with the new statistics or profile entries.
5. **Response Generator**: Returns a user-friendly response indicating the successfully logged information.

## Schema Definitions
- **Newborn Profile**: `name`, `birth_date` (optional), `initial_weight` (optional).
- **Feeding Log**: `type` (breastfeeding, formula, combination), `amount_oz` (optional), `timestamp`.
- **Diaper Log**: `type` (wet, dirty, both), `timestamp`.
- **Weight Log**: `weight_lbs_oz`, `timestamp`.

## Constraints & Safety Rules
- If not sure about which newborn is referred to, the agent **MUST** yield a `RequestInput` to clarify.
- Only accept valid numbers for feeding amounts and diaper counts.
- Keep data in-memory within the session.

## Success Criteria
- Accurately tracks multiple newborns.
- Clarifies newborn name when ambiguous.
- Logs daily stats (weight, feeding, diaper) correctly and updates the state.
